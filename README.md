# IMDB Redesign
Dette prosjektet går ut på å redesigne IMDB nettsiden ved å bruke min egen stil og farger. Jeg skal hente inn filminformasjon for nettsiden min fra TMDb API.
Målet er å lage en enklere, mer visuelt fokusert filmplattform hvor brukeren kan:
- Søke etter filmer og serier
- Se detaljer som poster, plot, skuespillere, rating og sjanger
- Utforske filmkort i et nytt UI-design
- Oppleve en renere og mer minimalistisk versjon av IMDB

Den første prototypen skal inneholde:

- Et søkefelt hvor man kan søke etter en film
- API-kall som henter filmdata fra TMDb API
- Vise resultat i et filmkort med:
  - Tittel
  - Poster
  - År
  - Sjanger
  - Rating
 - En enkel layout med HTML, CSS og JS

Når MVP fungerer kan jeg utvide med:

 - Flere søkeresultater samtidig
 - Favorittliste (lagret i LocalStorage)
 - Forbedret UI og animasjoner
 - Light/dark mode

Språk, teknologier og utstyr

Frontend:

- HTML
- CSS
- JavaScript
- Fetch API
- Figma

Backend (Planlagt Utvidelse)

- Flask (Python)
- MariaDB

Utviklingsverktøy

- VS Code
- Git/Github
- Figma

API 

- TMDb API – brukes for å hente filmtittel, skuespillere, poster, plot, år, sjanger og rating.

# Hvordan prosjektet viser kompentase i fagene

Utvikling

- Programmering: Lage nettsiden med HTML, CSS og JavaScript, og eventuelt backend med Flask.
- Debugging: Feilsøke API-kall, layout, datahåndtering og funksjonalitet.
- Brukersentrering: Redesigne IMDB med fokus på enkel navigasjon, modernisert UI og god brukeropplevelse.
- Versjonshåndtering: Bruke Git og GitHub til å lagre og utvikle prosjektet systematisk.
- Testing: Teste søkefunksjon, API-integrasjon og brukergrensesnitt for å sikre stabil funksjonalitet.

Driftstøtte 

 - Verktøybruk: Bruke utviklingsverktøy som Git, VS Code, Figma og API-verktøy.
 - Planlegging & dokumentasjon: Planlegge prosjektet, lage MVP, og dokumentere løsningene i Github Project.
 - Nettverk / Infrastruktur: Hente data fra eksterne API-er over HTTP.
 - Personvern: Sikre at API-nøkler og passord håndteres riktig
 - Databaser: (Utvidelse) Bruke MariaDB til å lagre brukere og favorittfilmer.
 - Automatisering: Forstå API-kall og programflyt som automatiserer innhenting av data.

Brukerstøtte

- Veiledning: Lage en tydelig og brukervennlig nettside som er enkel for brukeren å forstå.
- IT og samfunn: Vurdere hvordan tjenester som IMDB påvirker brukere og hvordan et bedre design gir bedre tilgjengelighet.
- Brukeradministrasjon: (Utvidelse) Håndtere brukere i et innloggingssystem med Flask + MariaDB.

Felles kompetanseområder

- Problemløsning: Finne løsninger når API-data mangler, når design må tilpasses, eller når funksjoner feiler.
- Dokumentasjon: README, planlegging, beskrivelser av systemet.
- Testing: Teste søkefunksjon, layout, API-kall, innlogging og responsivitet.
